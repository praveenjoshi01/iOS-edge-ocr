       - _initCamera(): call `availableCameras()`, pick `cameras.first` (back camera), create `CameraController` with `ResolutionPreset.high` (720p -- saves memory, ImagePreprocessor resizes to 1024px anyway) and `enableAudio: false` (OCR app, no microphone needed)
       - Initialize controller, setState on success
       - Handle camera init errors (no cameras found, permission denied)

       **Lifecycle management (CRITICAL -- Pitfall 1 from research):**
       - `didChangeAppLifecycleState`: On `inactive` -> dispose controller and set to null. On `resumed` -> re-initialize camera.
       - On dispose: remove observer, dispose controller
       - This prevents the documented iOS memory leak (Flutter issues #29586, #97941)

       **Permission handling:**
       - Before initializing camera, use PermissionService to check/request camera permission
       - If denied: show message explaining why camera access is needed
       - If permanently denied: show dialog with "Open Settings" button that calls `PermissionService().openSettings()`
       - Do NOT silently fail -- always tell user what happened (Pitfall 2)

       **Capture:**
       - Full-screen `CameraPreview(controller)` as the viewfinder
       - Large circular capture button at bottom center (FloatingActionButton.large with camera icon)
       - Guard against double-tap with `_isTakingPicture` flag
       - On capture: `await controller.takePicture()` returns XFile
       - Navigate to preview: `context.push('/preview?path=${Uri.encodeComponent(xFile.path)}')`
       - Show CircularProgressIndicator on capture button while taking picture

       **Layout:**
       - Stack: CameraPreview fills entire screen, capture button positioned at bottom center
       - AppBar with back button overlaid on camera preview (transparent background)
       - When controller not ready: Center with CircularProgressIndicator
       - When permission denied: Center with error message and Open Settings button

    2. **lib/features/ocr/presentation/ocr_view_model.dart** -- Add a new method `extractFromPath(String imagePath)` that accepts a pre-selected image path (from the preview screen) and runs the OCR pipeline:
       ```dart
       /// Extract text from a pre-selected image path.
       /// Called from PreviewScreen after user taps "Extract Text".
       /// Unlike pickAndExtract() which opens the gallery, this skips
       /// the picking step and goes straight to preprocessing.
       Future<void> extractFromPath(String imagePath) async {
         state = const OcrState.preprocessing();
         try {
           final runtimeNotifier = ref.read(edgeVedaRuntimeProvider.notifier);
           final ocrService = OcrService(runtime: runtimeNotifier);
           state = const OcrState.inferring();
           final result = await ocrService.extractText(imagePath);
           state = OcrState.complete(result);
         } catch (e) {
           state = OcrState.error(e.toString());
         }
       }
       ```
