       - Include part directive for generated file

    2. Create `lib/features/ocr/domain/ocr_state.dart`:
       - Freezed sealed union `OcrState` with states: idle, pickingImage, preprocessing, inferring, complete(OcrResult result), error(String message)
       - Include part directive for generated file

    3. Create `lib/core/constants/ocr_prompts.dart`:
       - Centralized prompt constants. Phase 1 only needs plain text.
       - `plainText`: 'Extract all visible text from this image. Return only the extracted text, preserving line breaks. Do not add any commentary.'
       - Keep prompts short and direct -- SmolVLM2 500M works best with simple, clear instructions.

    4. Create `lib/features/ocr/data/prompt_builder.dart`:
       - Class `PromptBuilder` with method `String buildPlainText()` that returns the plain text prompt from ocr_prompts.dart
       - This abstraction exists so Phase 3 can add structured/markdown/key-value prompt strategies without changing OCRService.

    5. Create `lib/features/ocr/data/image_preprocessor.dart`:
       - Class `ImagePreprocessor` with static method `Future<ProcessedImage> prepare(String imagePath)`
       - `ProcessedImage` class with fields: rgbBytes (Uint8List), width (int), height (int)
       - Implementation runs in background isolate via `compute()`:
         a. Read image bytes from file path
         b. Decode image using `package:image` (handles JPEG, PNG, HEIC)
         c. Apply EXIF orientation correction via `bakeOrientation()`
         d. Resize: if longest edge > 1024px, resize maintaining aspect ratio (cap at 1024px longest edge). Use `Interpolation.linear`.
         e. Convert to interleaved RGB bytes: R0G0B0R1G1B1... (3 bytes per pixel, NO alpha channel)
         f. Verify: rgbBytes.length == width * height * 3
         g. Return ProcessedImage

       CRITICAL MEMORY CONSTRAINT: Always resize to max 1024px longest edge. On iPhone 13 (4 GB RAM), full 12 MP images (48 MB raw) will push past iOS jetsam limits when combined with the ~600 MB model.

       CRITICAL FORMAT: Edge-Veda VisionWorker.describeFrame expects interleaved RGB (3 bytes/pixel). The Dart `image` package pixel accessor returns RGBA. Extract R, G, B channels only -- skip alpha.

    6. Run `dart run build_runner build --delete-conflicting-outputs` to generate freezed code.
  </action>

    - ImagePreprocessor.prepare compiles (uses compute, image package correctly)
    - RGB byte conversion logic: output length == width * height * 3

    1. Create `lib/features/ocr/data/ocr_service.dart`:
       - Class `OcrService` that takes EdgeVedaRuntimeNotifier reference
       - Method `Future<OcrResult> extractText(String imagePath)`:
         a. Record start time (Stopwatch)
         b. Preprocess: `ImagePreprocessor.prepare(imagePath)` -- runs in isolate
         c. Build prompt: `PromptBuilder().buildPlainText()`
         d. Infer: call runtime's `describeFrame(processed.rgbBytes, processed.width, processed.height, prompt: prompt, maxTokens: 1024)`
         e. Post-process result text: trim whitespace, remove any obvious artifacts
         f. Return `OcrResult(text: result.text, processingTimeMs: stopwatch.elapsedMilliseconds, imageWidth: processed.width, imageHeight: processed.height)`
       - This class enforces the service-mediated inference pattern: UI never calls VisionWorker directly.

    2. Create `lib/features/ocr/presentation/ocr_view_model.dart`:
       - Riverpod AsyncNotifier `OcrViewModel` (use @riverpod annotation)
       - State is `OcrState` (from domain/ocr_state.dart)
       - build() returns OcrState.idle
       - Method `pickAndExtract()`:
         a. Set state to OcrState.pickingImage
         b. Use `image_picker` package (add to pubspec.yaml if not present) to pick image from gallery: `ImagePicker().pickImage(source: ImageSource.gallery)`
         c. If user cancelled, return to OcrState.idle
         d. Set state to OcrState.preprocessing
         e. Get OCRService (create with runtime ref)
         f. Set state to OcrState.inferring
         g. Call ocrService.extractText(pickedFile.path)
         h. Set state to OcrState.complete(result)
         i. On error: set state to OcrState.error(e.toString())
       - Method `reset()`: returns to OcrState.idle

       NOTE: Add `image_picker: ^1.1.2` to pubspec.yaml dependencies. This is needed for the test screen to pick images. It will also be used in Phase 2 for the full image input flow.

       - This is a minimal test screen. Phase 2 replaces it with the full image input flow. Phase 3 adds loading polish and error handling. For now, functional correctness is the goal.
