  <action>
    1. Create `lib/runtime/model_downloader.dart`:
       - Class `ModelDownloader` with method `Stream<DownloadProgress> download(String url, String destPath)`
       - `DownloadProgress` class with fields: downloadedBytes (int), totalBytes (int), percentage (double 0.0-1.0)
       - Resume support: check if `$destPath.tmp` exists, get its length, send HTTP Range header `bytes=$existingLength-`
       - Write chunks to `.tmp` file, yield progress after each chunk
       - After complete download, rename `.tmp` to final path (atomic rename)
       - Use `dart:io` HttpClient (not the http package) for Range header support
       - Handle edge cases: server doesn't support Range (re-download from 0), content-length unknown (report bytes without percentage)
       - Method `downloadAll()` that downloads BOTH model + mmproj files sequentially, yielding combined progress

    2. Create `lib/runtime/edge_veda_runtime.dart`:
       - Riverpod AsyncNotifier `EdgeVedaRuntimeNotifier` (use @riverpod annotation)
       - build() method: check ModelConfig.isModelReady. If ready -> spawn VisionWorker, initVision with modelPath, mmprojPath, numThreads: 4, contextSize: 2048, useGpu: true, return RuntimeState.ready(). If not ready -> return RuntimeState.uninitialized().
       - `downloadModel()` method: creates ModelDownloader, streams progress for both files, updates state with RuntimeState.downloading(...) on each progress event. After download complete, calls initVision and transitions to RuntimeState.ready().
       - `describeFrame()` method: delegates to VisionWorker.describeFrame(). Throws StateError if worker is null.
       - `dispose()` via ref.onDispose: calls worker.dispose() to clean up
       - Import edge_veda package for VisionWorker, FrameDescription types

    3. Create `lib/features/onboarding/presentation/download_view_model.dart`:
       - Riverpod provider that watches EdgeVedaRuntimeNotifier state
       - Exposes methods: startDownload(), retryDownload()
       - Derives display values: progressPercent (formatted string), downloadedMB/totalMB, isDownloading, isReady, hasError

    4. Create `lib/features/onboarding/presentation/download_screen.dart`:
       - ConsumerWidget that watches download_view_model
       - States:
         - Uninitialized: Show "Download AI Model" heading, model size info (~437 MB + mmproj), Wi-Fi recommendation, "Download" button
         - Downloading: Show circular progress indicator with percentage, "X MB / Y MB downloaded" text, no cancel needed for Phase 1
         - Initializing: Show "Loading AI model..." spinner
         - Ready: Show "Ready!" with checkmark, auto-navigate or show "Continue" button
         - Error: Show error message with "Retry" button
       - Clean, centered layout. No fancy animations. Functional first.

    5. Update `lib/app.dart` to use go_router with proper routing:
       - '/' route: Check runtime state. If ready -> navigate to OCR test screen (placeholder for now). If not -> show DownloadScreen.
       - '/ocr' route: Placeholder screen with text "OCR Ready - Phase 1 Plan 02 will build this"

    6. Run `dart run build_runner build --delete-conflicting-outputs` to generate Riverpod provider code.

    CRITICAL: VisionWorker lifecycle from Edge-Veda docs: spawn() -> initVision() -> describeFrame() -> dispose(). spawn() must be called before initVision(). initVision() requires BOTH modelPath AND mmprojPath.

    NOTE: For the download screen, keep it simple and functional. The user needs to see progress. No need for elaborate design in Phase 1.
